◆ current_block_energy_mode()

uint32_t current_block_energy_mode ( void  )

Returns the highest energy mode that has a block on it.

This function is the only way to read the private lowest_energy_mode array outside of sleep_routines.c.

Note
In this implementation, MAX_ENERGY_MODES is defined in sleep_routines.h and must be changed if the file is included to control a system with more energy modes.
Returns
The highest energy mode with a block on it as an integer.

◆ enter_sleep()

void enter_sleep ( void  )

Sets the system to the lowest sleep mode that peripherals will allow.

This function depends on the lowest_energy_mode array and definitions in sleep_routines.h.

Note
If this function is not functioning as expected, ensure that sleep_open() and sleep_unblock_mode are creating the lowest_energy_mode array.

◆ sleep_block_mode()

void sleep_block_mode ( uint32_t  EM)

Sets the sleep mode at which the system should not go below.

This function changes the value in the lowest_energy_mode array, which is later used to prevent a specific sleep mode.

Note
The EFM_ASSERT() line is placed in order to prevent the system from getting stuck in a loop and setting the same array index for too long.
Parameters
[in]EMThe EM parameter should hold the value of the Energy Mode that should be blocked.

◆ sleep_open()

void sleep_open ( void  )

Enables the sleep blocking system.

This function initializes the lowest_energy_mode array in order to prevent the system from entering a sleep mode that would disable a necessary peripheral.

Note
This function should be called before any other sleep-blocking function.

◆ sleep_unblock_mode()

void sleep_unblock_mode ( uint32_t  EM)

Removes a blocked sleep mode.

This function changes the value in the lowest_energy_mode array to remove a block previously placed.

Note
This function must be called on the completion of whatever called the previous sleep_block_mode().
Parameters
[in]EMThe EM parameter should hold the value of the Energy Mode that should be unblocked.
